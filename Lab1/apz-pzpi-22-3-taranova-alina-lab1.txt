BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1 Background (Передумови)
Знання іноземних мов є важливим інструментом у глобалізованому світі,
оскільки вони відкривають доступ до нових можливостей у навчанні, кар’єрі та
міжкультурній комунікації. Проте процес їх вивчення часто залишається викликом
для багатьох людей через одноманітність методів, брак мотивації та складність
запам’ятовування великої кількості нової лексики. Традиційні методи, такі як
читання словників, використання підручників чи флеш-карток, не завжди
забезпечують ефективне закріплення знань і швидке оволодіння мовою.
Успіх вивчення залежить від активного застосування нових слів у контексті
та регулярного повторення. Інтерактивні вправи, зокрема кросворди, створюють
додаткову мотивацію до навчання, оскільки поєднують гру з навчанням,
стимулюючи користувача до більш активної участі. Створення застосунку, що
поєднує ці підходи, може не тільки зробити процес вивчення мов легшим і
цікавішим, але й значно підвищити ефективність запам’ятовування нових слів,
забезпечуючи довготривале засвоєння знань.
1.2 Business Opportunity (Бізнес-можливості)
Ринок мобільних застосунків для вивчення мов активно розвивається. Багато
програм пропонують флеш-картки та інші методи для вивчення лексики, але не
кожен з них використовує інтерактивний підхід, який сприяє ефективнішому
засвоєнню матеріалу. У цьому проекті поєднуються традиційні методи з новими
Ринок мобільних застосунків для вивчення мов активно розвивається, і потреба в
ефективних і захоплюючих рішеннях зростає. Сучасні користувачі очікують, що
продукти для навчання будуть не тільки корисними, але й інтуїтивно зрозумілими,
інтерактивними та доступними. Проект пропонує унікальну можливість заповнити
прогалину між традиційними методами навчання та сучасними технологіями.
Особливість продукту полягає у використанні інтерактивних кросвордів як
основного засобу навчання. Також застосунок впроваджує інтервальне повторення,

Таким чином, наша система не тільки враховує переваги конкурентів, але й

VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

2.1 Vision Statement (Окреслення концепції)
Проект має створити потужний та зручний back-end для системи вивчення
лексики і головоломок, який забезпечить:
- Безпечну реєстрацію та JWT-авторизацію користувачів (bcrypt + JWT).
- CRUD-операції для флеш-карток та кросвордів через REST API, з повною
документацією у Swagger UI.
- Інтелектуальне планування інтервальних повторень за алгоритмом SM-2.
- Аналітичні метрики:
- TF–IDF оцінка важливості тегів у наборах флеш-карток.
- «Крива забування» (retention curve) — ймовірність відтворення слова з
урахуванням часу.
- Рольову модель з розподілом прав **user** / **admin**, що дозволяє
адміністраторам:
- Переглядати та видаляти користувачів.
- Змінювати ролі користувачів.
- Отримувати звітну статистику (загальна кількість користувачів, карток,
середнє на користувача).
- Швидке розгортання і локальну розробку за допомогою Docker та dockercompose.
2.2 Major Features (Головна функціональність)
Продукт пропонує широкий спектр функцій, включаючи створення
інтерактивних кросвордів, вправ для відновлення визначень та інтеграцію
інтервального повторення. У кросвордах користувачі зможуть вирішувати
завдання, засновані на вивчених словах, що сприяє їх закріпленню в контексті.
Вправи з відновлення визначень допоможуть розвивати мовні навички та
тренувати пам’ять.

4.1 Operating Environment (Робоче середовище)
Програмні платформи, що будуть використані у розробці усіх частин
програмної системи, включають: Java SE platform; Python 3 interpreter; Docker разом
з Docker Compose. СКБД: PostgreSQL. Фреймворки: Java OkHTTP; Python FastApi
framework; Android SDK; та інші. Мови програмування, що будуть використані під
час розробки: Java; Python 3; PL/SQL; скрипти Dockerfile.
Для розгортання серверної частини у Docker-контейнерах було обрано
хмарний сервіс DigitalOcean.
Інтегровані середовища програмування, що використовуватимуться у
процесі розробки: Android Studio; PyCharm; Visual Studio Code.
Основні вимоги до програмної системи:
1. Доступність. Інтерфейс Android-застосунку повинен бути простим у
користуванні та інтуїтивно зрозумілим;
2. Надійність. Стабільне з’єднання з мережею;
3. Продуктивність. Основна вимога - сервер має бути здатним обслуговувати
до одного десятку тисяч клієнтів одночасно;
4. Цілісність. Шифрування даних під час подорожі з клієнту на сервер.
Хешування паролів та особистих даних користувачів на сервері. Використання
JWT-аутентифікації між клієнтом та сервером, а також сесій.
